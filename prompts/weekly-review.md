Read the following files:
- `data/today.json` (for this morning's biometrics)
- `training/plan.md` (for the current training block context)
- All reports from `reports/` for the past 7 days (YYYY-MM-DD.md)

Then produce the weekly review in exactly this structure:

---

## Weekly Training Review — Week of {{WEEK_START}} to {{WEEK_END}}

**Phase:** [Base / Build / Peak / Taper / Recovery]
**Plan Week:** [N of 16]

---

### 1. WEEK SUMMARY

High-level narrative: what kind of week was it? (2–3 sentences)

### 2. VOLUME & LOAD

| Metric | Actual | Target | Status |
|--------|--------|--------|--------|
| Total distance | X km | Y km | ✅/⚠️/❌ |
| Total duration | X h | Y h | ✅/⚠️/❌ |
| Weekly TSS | X | Y–Z | ✅/⚠️/❌ |
| Long run | X km | Y km | ✅/⚠️/❌ |

### 3. ZONE DISTRIBUTION

| Zone | % of time | Assessment |
|------|-----------|------------|
| Z1-2 (easy) | X% | [good if > 80%] |
| Z3 (moderate) | X% | [flag if > 15%] |
| Z4-5 (hard) | X% | [flag if > 10% without intent] |

### 4. RECOVERY TREND

Weekly averages:
- **HRV:** [avg] ms (vs previous week: [+/-X]%)
- **Sleep:** [avg]h per night, avg score [score]/100
- **Body Battery:** avg [value]/100 on waking
- **Resting HR:** avg [value] bpm

### 5. TRAINING EFFECTIVENESS

Were the key sessions executed as planned? Was the quality there?
Note any sessions that were skipped, shortened, or modified.

### 6. NEXT WEEK GUIDANCE

What should next week look like? Reference the training plan and current fatigue state.
State any adjustments to the planned sessions concretely.

### 7. PLAN ANNOTATION (if adjustment warranted)

If a plan adjustment is recommended, write the exact annotation to add to training/plan.md
using the format: `<!-- AI [YYYY-MM-DD]: [recommendation] -->`

Only annotate if the data clearly warrants it. No annotation = healthy training, plan stands.
